ETISS Configuration/Options (runtime)

This page list options that can be passed to ETISS or set at runtime.

Options are parsed by etiss::initialize() and can be accessed/changed with etiss::cfg(). The following table contains a list of current options. Plugins may have theire own options. Please have a look at the respective plugin documentation for auch options. The documentation for the integrated library and related plugin implementations can be found Implementation Status here.

Option
Values Description
loglevel SILENT = 0
FATALERROR = 1
ERROR = 2
WARNING = 3 (default)
INFO = 4
VERBOSE = 5

sets the log level an suppresses messages of higher log levels.
e.g. "-floglevel=5" or "-ologlevel 5"
etiss_wd path etiss_wd must conatain a valid path (or a path that can be created with mkdir) where the program has read/write permission. By default the path "~/.etiss/<ETISS_VERSION_INTERFACE>/" is used. The ~/ path is resolved using the HOME environment variable (see getenv("HOME"))

Attention: etiss_wd can only be set with the arguments passed to etiss::initialize. Later changes are not possible and calls to etiss::cfg().set("etiss_wd",...) will be ignored.
copy-headers true/false This option allows ETISS to writing its header files into the etiss_wd folder. this option is enabled by default and should not be disabled.
sigint-console true/false (default) if enabled then pressing ctrl-c will stop ETISS and a python console can be used. this option is experimental and currently doesn't support mutithreading. use with caution
library-loading true (default)/false if enabled then ETISS will try to locate plugin libraries and load them. if not enabled then external libraries must be loaded manually.
integrated-library true (default)/false if enabled then the integrated plugin library (gdb server,logger, etc. [see Implementation Status]) of ETISS will be enabled.
pyconsole true/false (default) if enabled then a python console will be provided after initialization has been done. Use ctrl-d to exit the console and return from the initialization function.
